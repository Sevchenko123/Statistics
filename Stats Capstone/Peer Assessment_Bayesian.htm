Median and interquartile range (IQR) are good summary statistics to use for least/most expensive and spread of housing price. The reason is that they are stable statistics that are not affected by extreme values (which mean and standard deviation are).

A boxplot is a good choice for displaying these statistics. The boxplot above displays median value (the horizontal black bar), first and third quartile (the box itself), smallest and largest non-outliers (endpoints of the dashed lines), as well as outliers (circles beyond the endpoints of the dashed lines).

The variation in house prices is displayed by the size of the boxplot. Blueste has the smallest boxplot, no outliers, and is therefore the neighborhood with the least variation in price. Blueste’s price range is 116500 to 137000. NridgHt has the most variation in price, with the range 173000 to 615000.

Most expensive/least expensive area is determined by the median of price, the higher the median the more expensive neighborhood, and vice versa. The least expensive area is MeadowV, with median price 85750. The most expensive area is StoneBr, with median price 340691.5.

We want to predict the natural log of the home prices. Candidate explanatory variables are lot size in square feet (Lot.Area), slope of property (Land.Slope), original construction date (Year.Built), remodel date (Year.Remod.Add), and the number of bedrooms above grade (Bedroom.AbvGr). Pick a model selection or model averaging method covered in the Specialization, and describe how this method works. Then, use this method to find the best multiple regression model for predicting the natural log of the home prices.

bma_model1 <- bas.lm(log(price) ~ Lot.Area + Land.Slope + Year.Built + Year.Remod.Add + Bedroom.AbvGr, data = ames_train, prior = "BIC", modelprior = uniform())
plot(bma_model1, which = 4, ask=FALSE, caption="", sub.caption="")

image(bma_model1, top.models = 10, rotate = F)

Bayesian Model Averaging (BMA) will be used as the base model to find the model for estimation. This is because BMA uses multiple models and represents the full posterior uncertainty for the data.

The BMA model is created with Bayesian Information Criterion (BIC) used as metric for model selection. BIC is based on model fit, while simultaneously penalizing the number of parameters in proportion to the sample size, so it is a good metric for finding a model with not too many parameters. Model prior is set as uniform, meaning all models have equal prior probability.

The first plot shows the marginal posterior inclusion probabilities for each of the variables. It looks like all variables are good predicators and are likely to be included in the final model. The “worst” predicator is Land.Slope with a 60% chance of being included in the final model.

The second plot shows model ranks and the top ranked model is the one including all variables, so that is the best model for predicting log(price).

I would think that the log transformation is better. The lot area distribution is very right skewed and much of the data cluster near zero and all observations are positive, so it’s a textbook example where log transformation can be used to make the relationship between the two variables more linear. (This is not shown here, but can be displayed by plotting histogram of lot area and log transformed lot area.)

Subtracting log transformed price from the respective predicted price from the two models and plotting histogram would show that the one that is mostly centered around zero is the best predicator. If a model was perfect, predicted price minus observed price would be zero for each prediction. Looking at the plots it’s very close, but the second plot with log transformed is slightly more centered around zero. The second model with log transformed is therefore probably better.
